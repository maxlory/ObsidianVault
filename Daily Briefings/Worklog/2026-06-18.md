## skill-fetch 深度评测与优化

- 验证 skillsmp-mcp-server 仓库，确认其 5 个 tool 中仅 1 个真正调 SkillsMP API，其余均为 curl/npx 壳层，结论：**不安装**，改用 skill-fetch + 匿名 API 直接搜索
- 逐一测试 skill-fetch 9 个搜索源可用性：SkillsMP AI 死掉、PolySkill 包损坏、ClawhHub 信噪比低 → 淘汰 3 源，最终保留 **7 可用源**
- 注册并配通 SkillHub API key，验证 Skills Directory 实际免 key 可用
- 用 skill-builder 评测体系跑 3 组对比测试（前端测试/LangGraph/调试模糊查询），with_skill 平均通过率 85%，发现 **API key 泄露** 严重安全漏洞
- 重新设计评分体系：新增 **Adoption（安装量）** 权重 + **三级渐进式内容深读（L1 扫描→L2 结构→L3 精读）** + 修复 API key 泄露、Trust 偏向等问题
- 迭代后通过率升至 **100%**，新评分的首次安装量信号立功（发现 jezweb/vitest 有 1135 安装、nousresearch/hermes-agent 有 196K stars 的 2377 行 SKILL.md）
- 与 find-skills 做 head-to-head 对比：断言通过率平手（均 100%），但 skill-fetch 独家找到 Anthropic 官方 mcp-builder、hermes-agent 等更高质量 skill
- 根据评测结果将 9 源精简为 **4 源**（skills.sh > GitHub > SkillsMP > Anthropic），删除 SkillHub/Skills Directory/ClawhHub 等低价值源
- 参照 find-skills 关键词策略，修复 skill-fetch 搜索关键词变体不足问题

## 开源仓库调研（ralph-tui + agentmemory）

- **ralph-tui**：调研发现是 AI 编程助手自动调度器，但**不原生支持按任务类型分配不同模型**（一 session 一模型），workaround 需拆两个 session + wrapper 脚本
- **agentmemory**：了解其 4 层记忆金字塔 + 三路混合搜索（BM25/向量/知识图谱）实现机制，默认 embedding 用本地 all-MiniLM-L6-v2（384 维），**无额外 token 消耗** 但关键功能（自动压缩/上下文注入）默认关闭
- **翻译工具调研**：实测 PDFMathTranslate、bilingual_book_maker 等均需 API key（Claude Max 套餐用不了），最终在 skills.sh 找到 **baoyu-translate**（19.6k 安装、21.8k star），直接安装即可用 Claude Code 做高质量翻译
