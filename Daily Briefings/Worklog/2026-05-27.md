## 技能数量膨胀与触发优化调研
- 调研确认：36 个 skill 的 description 每轮固定烧 ~32K token，是 token 暴涨和 agent 少调用的根因。候选越多、description 越重叠，选错/漏选越严重
- 找到官方解法：Tool Search（defer_loading）实测省 85% token 且准确率从 49%→74%；以及 skillOverrides 禁用、name-only 等 4 种瘦身开关
- 深入调研 name/description 写作优化，发现关键：description 要 pushy、写负向边界（Do NOT use when）、控制单条 ≤500 字符防被全局预算静默丢弃；官方 skill-creator 自带评测脚本可量化触发率
- 将调研成果整理为 memory 文件 `name_description_authoring.md`，并用自己的标准自检后做了修订
